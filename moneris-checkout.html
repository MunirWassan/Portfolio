Moneris Checkout & Hosted Tokenization Demo

Use this page to spin up a QA payment session against Moneris using their hosted solutions and capture the responses returned to your site.

1. Configure your API credentials

Store your store ID and API token as environment variables on the server that will call Moneris. The sample Express server in server/moneris-checkout-server.js expects MONERIS_STORE_ID, MONERIS_API_TOKEN, and optionally MONERIS_ENV (defaults to qa).

2. Start the demo helper API

Install dependencies with npm install and start the sample server with npm run moneris:server. The server hosts this page on http://localhost:5174/moneris-checkout.html and exposes a POST /api/moneris/checkout endpoint that proxies the Create Checkout request documented by Moneris. Update the payload or headers there if you need to add optional fields such as billing details.

3. Launch Moneris Checkout from this page

Enter the amount you want to authorize and click Request Checkout Session. The helper API returns a checkout ID that the Moneris JavaScript SDK uses to open the hosted experience. When the payment flow finishes the page logs the response payloads below.
